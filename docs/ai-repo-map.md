# AI Repo Map

This file is optimized for fast repository navigation by AI tools and humans making targeted changes.

## Repository Purpose
This repository is an async Python trading bot centered on Polymarket binary markets. Its main implemented loop is:

1. discover markets
2. fetch/order-book updates
3. build `MarketState`
4. detect bundle arbitrage or market-making opportunities
5. pass signals through risk checks and execution
6. track fills, positions, exposure, and PnL
7. optionally publish state to a live dashboard

The repo also includes Kalshi market loading, text-based matching, and cross-platform arbitrage logic, but cross-platform execution is not wired into the live bot loop.

## Main Runtime Paths
### Polymarket bot path
`main.py`

- loads config
- builds `PolymarketClient`, `Portfolio`, `RiskManager`, `ExecutionEngine`, `ArbEngine`, `DataFeed`
- subscribes `TradingBot._on_market_update()` as the feed callback
- submits generated signals to `ExecutionEngine`
- runs a monitoring loop for snapshots and risk PnL updates

### Dashboard path
`run_with_dashboard.py`

- does the same Polymarket setup as `main.py`
- adds `DashboardIntegration`
- starts `uvicorn` for `dashboard.server:app`
- optionally creates `KalshiClient` and a `CrossPlatformArbEngine`
- loads Kalshi markets and runs `MarketMatcher.find_matches()`

## Important Behavior Notes
### Live order book ingestion
The primary "real" Polymarket stream path is in `PolymarketClient.stream_orderbook()` and is polling-based. It rotates through a batch of markets and fetches YES and NO books via REST. The WebSocket helper exists but is not the active production path.

### Dry run
Dry-run order placement, open order state, and simulated fills live inside `PolymarketClient`. `main.py` and `run_with_dashboard.py` optionally start a background fill simulation loop.

### Risk volume filter
`RiskManager` supports volume-based gating, but the main bot does not populate volume data into the risk manager in production flow. If `trade_only_high_volume` is enabled without volume population, orders will be rejected.

### Cross-platform
`CrossPlatformArbEngine.check_arbitrage()` exists, but the current dashboard bot path does not call it from live market updates.
